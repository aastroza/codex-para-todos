# Prompts para las demos del taller

Estos prompts están pensados para ejecutarse dentro del repositorio
[`idsudd/codex-para-data-science`](https://github.com/idsudd/codex-para-data-science).

Antes de cada demo, abre el repositorio en Codex y confirma que puede leer
`PROBLEMA.md` y `AGENTS.md`. Los prompts preservan el test oficial y piden
evidencia reproducible en vez de una respuesta aislada en el chat.

## Demos

1. [EDA visual como micrositio](01-eda-visual.md)
2. [Un agente construye y otro critica](02-constructor-y-revisor.md)
3. [Herramienta de anotación con `/goal`](03-herramienta-anotacion-goal.md)

Los textos son puntos de partida. Durante la demo conviene detenerse después
del plan, revisar las decisiones con el grupo y recién entonces autorizar la
implementación.
